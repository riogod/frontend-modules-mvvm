# Генератор модулей для нагрузочного теста

## Описание

Скрипт `generate-load-test-modules.mjs` создает 100 MFE модулей для нагрузочного тестирования системы. Каждый модуль содержит минимум 20 DI (Dependency Injection) сущностей с взаимными зависимостями.

## Структура модулей

Каждый модуль (`load-test-001` до `load-test-100`) содержит:

- **6 Models** - MobX модели с состоянием
- **3 ViewModels** - View-модели с зависимостями от Models и Usecases
- **8 Usecases** - Use cases с зависимостями от Models и других Usecases
- **2 Repositories** - Репозитории для работы с данными
- **2 Services** - Сервисы с зависимостями от Repositories и других Services

**Итого: 21 DI сущность на модуль**

## Зависимости

Сущности имеют взаимные зависимости:
- ViewModels зависят от Models и Usecases
- Usecases зависят от Models и других Usecases
- Services зависят от Repositories и других Services

Это создает сложный граф зависимостей для тестирования DI контейнера.

## Использование

```bash
node scripts/generate-load-test-modules.mjs
```

Скрипт:
1. Создает базовую структуру модуля через `ModuleGenerator`
2. Генерирует все DI сущности с зависимостями
3. Обновляет `di.tokens.ts` и `di.config.ts`
4. Создает index файлы для экспорта

## После генерации

После генерации модулей необходимо:

1. Добавить модули в `host/src/modules/modules.ts` (опционально, для автоматической загрузки)
2. Установить зависимости: `npm install --legacy-peer-deps`
3. Запустить сборку для проверки: `npm run build`

## Очистка

Для удаления всех сгенерированных модулей:

```bash
rm -rf packages/load-test-*
```

## Примечания

- Модули создаются с префиксом `load-test-` и нумерацией от 001 до 100
- Все сущности используют префикс `entity` для именования
- Зависимости создаются случайным образом, но с ограничением максимум 3 зависимости на сущность
- Скрипт показывает прогресс каждые 10 модулей
