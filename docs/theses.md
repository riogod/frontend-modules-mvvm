## Варианты запуска лаунчера (dev) и поставка данных

- `dev:direct` (`vite --config host/vite.config.mts`): манифест собирается локально (`VITE_LOCAL_MODULES` не указан) — remoteEntry из манифеста используются как есть (если пусто — алиас/HMR).
- `dev:all-local` (`VITE_LOCAL_MODULES=*`): все модули принудительно локальные; remoteEntry в манифесте игнорируются, загрузка через alias/HMR.
- Лаунчер с `configs.json`: если выбран `remote` manifest URL — подтягивается удалённый манифест; далее:
  - модули с `remoteEntry=""` → локальные alias/HMR;
  - модули с `remoteEntry` (URL) → загрузка скрипта и статики с указанного origin;
  - отключённые в UI/фильтре модули должны быть вырезаны/переопределены, иначе будут грузиться по своим remoteEntry (текущая проблема: подмена на 4200).
